한 줄 결론

Autopilot의 핵심 가치는 "더 많이"가 아니라 "더 빨리" 판매 + 셀러 재방문입니다. 거래완료·첫 채팅이 약 2배 빠르고 재등록률이 3배 높지만, 장기 누적 판매율은 수동 끌올이 약간 높습니다(끌올은 오래 걸려도 결국 더 팔림).

05왜 Autopilot 판매율이 더 낮은가

autopilot의 약점이 아니라, 붙는 물건의 가격 구성 + 비교군(끌올) 선택편향

구성Autopilot끌올 only
평균 가격$17,291$9,461
중앙값 가격$40$30
무료(나눔) 비율1.6%3.6%
Autopilot은 자동 가격인하 특성상 더 비싼 물건에 붙고 나눔 비중은 절반입니다. 비싼 물건은 원래 잘 안 팔리고 나눔은 거의 다 팔리므로, autopilot 쪽 모수가 구조적으로 불리합니다.

가격대별 판매율 — 통제하면 격차가 사라짐

$50 이상에선 사실상 동률. 전체 격차는 대부분 "비싼 물건 집중" 구성효과

Autopilot 끌올 only
저가·나눔 구간에서만 끌올이 앞서는데, "끌올 only"는 정의상 셀러가 직접 손으로 끌어올린 글이라 적극적 셀러로 선택편향이 있습니다($5짜리를 수동 끌올하는 사람은 잘 팖). Autopilot은 default ON이라 소극적 셀러까지 포함됩니다.
프로모션 부족은 아님. Autopilot 글의 72%가 수동 끌올도 함께 했고 평균 끌올 횟수도 비슷합니다(3.4 vs 3.5회). 덜 노출돼서 안 팔린 게 아닙니다. → 같은 가격대에선 동등하고, 속도는 모든 구간에서 autopilot이 빠릅니다.

07Default ON & OFF 전환

글쓰기 시 autopilot이 default ON으로 노출

명시적으로 OFF로 끄는 토글은 주당 약 6,300~7,300 세션(월 ~2.8만)으로 기간 내 안정적입니다. 토글을 만진 세션 중 약 60%가 OFF로 끝납니다. 전체 게시글 기준으로는 31%가 ON 유지, 약 7%가 명시적 OFF, 나머지는 비대상(나눔·특정 카테고리 등)으로 추정됩니다.
로깅 갭. 가격제안 화면 노출 이벤트(client_shown_fleamarket_writePriceSuggestion)에 write_session_id가 비어 있어 "노출 대비 OFF율"을 세션 단위로 정확히 산출할 수 없습니다. 정밀 측정을 위해 노출 이벤트에 세션 ID 적재가 필요합니다.

08사용된 쿼리

정의 — autopilot = dynamic_price_infos 보유 (판매율·소요시간) 
-- autopilot = 자동인하 설정 보유 게시글 (CA)
WITH autopilot_articles AS (
  SELECT DISTINCT CAST(article_id AS STRING) aid
  FROM `karrotmarket.db_fleamarket_ca.dynamic_price_infos`
  WHERE DATE(created_at) BETWEEN '2025-05-01' AND '2026-06-16'
),
republish_articles AS (
  SELECT DISTINCT CAST(article_id AS STRING) aid
  FROM `karrot-data-production.karrotanalytics_v1_ca.server_republished_fleamarket_article_v1`
  WHERE DATE(event_timestamp) BETWEEN '2025-05-17' AND '2026-05-15' AND article_id IS NOT NULL
),
base AS (
  SELECT CAST(article_id AS STRING) aid, created_at, reserved_at, closed_at, buyer_hoian_user_id, hoian_user_id
  FROM `karrot-data-production.dbt_dimension_ca.fleaMarketArticles`
  WHERE DATE(created_at) BETWEEN '2025-05-17' AND '2026-05-15'
    AND hoian_user_id NOT IN (/* 내부 유저 제외 */)
),
g AS (
  SELECT CASE WHEN a.aid IS NOT NULL THEN 'AUTOPILOT' WHEN r.aid IS NOT NULL THEN 'REPUBLISH_ONLY' END grp,
    (closed_at IS NOT NULL AND buyer_hoian_user_id IS NOT NULL) is_sold,
    IF(closed_at IS NOT NULL AND buyer_hoian_user_id IS NOT NULL, TIMESTAMP_DIFF(closed_at,created_at,HOUR)/24.0, NULL) d_close
  FROM base b LEFT JOIN autopilot_articles a ON b.aid=a.aid LEFT JOIN republish_articles r ON b.aid=r.aid
  WHERE a.aid IS NOT NULL OR r.aid IS NOT NULL
)
SELECT grp, COUNT(*) n, COUNTIF(is_sold) sold_count,
  APPROX_QUANTILES(d_close,100)[OFFSET(50)] p50_close
FROM g GROUP BY grp;
규모·유저 — 일평균 신규글·autopilot%·유저수·DAU / 설정 분포(30일 제한) 
-- 일평균 신규글 + autopilot ON 비율
SELECT COUNT(*) total, COUNTIF(ap.aid IS NOT NULL) ap_on,
  COUNT(*)/(DATE_DIFF(DATE '2026-05-15', DATE '2025-05-17', DAY)+1) daily_avg
FROM ca_articles a LEFT JOIN autopilot_articles ap USING(aid);

-- autopilot 유저수 + DAU 대비 (visited_app)
-- autopilot_users / 활성유저(period unique) = 15.5%, / avg DAU = 4.6배

-- 설정 분포: 판매기한 0~30일 + Free(최저가=0) 비율
WITH d AS (
  SELECT article_id, ARRAY_AGG(STRUCT(percent_discount, minimum_price, ends_at, created_at)
           ORDER BY created_at DESC LIMIT 1)[OFFSET(0)] r
  FROM `karrotmarket.db_fleamarket_ca.dynamic_price_infos`
  WHERE DATE(created_at) BETWEEN '2026-01-01' AND '2026-06-15' AND ends_at IS NOT NULL AND original_price>0
  GROUP BY article_id
)
SELECT COUNTIF(mp=0)/COUNT(*) free_floor_pct, ...
FROM (SELECT r.minimum_price mp, TIMESTAMP_DIFF(r.ends_at,r.created_at,DAY) h FROM d)
WHERE h BETWEEN 0 AND 30;
해석 주의. ① autopilot은 default ON이라 "설정 보유"가 적극 선택을 뜻하진 않음 → 판매율이 끌올보다 낮은 건 끌올이 "셀러가 적극 푸시한 글"이라는 선택편향 영향이 큼. 가장 신뢰도 높은 우위는 속도(②⑤)와 재등록(⑦). ② 설정값·침투율은 CA 전용(dynamic_price_infos). US는 원천 테이블 미확인. ③ 지표 ③④와 채팅·설정 분포는 2026 코호트, 나머지는 전기간.
